# Wireframes and UX Flows

## 1. Merchant Onboarding Flow

### Screen 1: Welcome / Intake Bot
- Greeting.
- What do you sell?
- How many products do you have?
- How do you currently receive orders?
- Biggest pain point?

### Screen 2: Value Pitch
- Personalized summary.
- Benefits of using Lameda.
- Button: See Demo.
- Button: Continue.

### Screen 3: Magic Link Activation
- Business name.
- Email.
- Create account.
- Confirm activation.

### Screen 4: Setup Wizard
- Connect channel.
- Upload products.
- Set delivery rules.
- Set return rules.
- Set FAQs.
- Choose persona.
- Go live.

## 3. Customer Browse Flow

### Chat Entry
- Customer taps bot link.
- Consent message appears.
- Menu shows Shop, Search, My Orders, Help.

### Browse Layout
- Category list.
- Product cards with image, price, stock, and CTA.
- Pagination.

### Search Layout
- Customer types request.
- Bot interprets intent.
- Top matches displayed.
- Option to refine search.

## 4. Product Detail Flow
- Product image.
- Description.
- Price.
- Size or color variants.
- Stock availability.
- Buttons: Add to Cart, Ask Question, Back.

## 5. Cart and Checkout Flow
- Cart summary.
- Delivery address capture.
- Delivery fee calculation.
- Payment option selection.
- Confirmation.

## 6. Complaint Flow
- Customer raises issue.
- Bot acknowledges and logs complaint.
- Merchant gets alert.
- Merchant chooses replace, refund, exchange, or investigate.
- Resolution is confirmed.

## UX Principles
- Keep messages short.
- Reduce form burden.
- Use buttons where possible.
- Escalate to humans when conversation quality drops.
- Make status changes visible and proactive.
